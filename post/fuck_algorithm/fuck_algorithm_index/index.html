DP

找到状态跟递推关系式(状态转移方程)最麻烦，往往由递归树由底向上递推。实在做不出来就写个递归，用集合保存过程结果。

树

树的问题都可以转化为前中后序遍历。

排序

O(n^2):插入、冒泡、选择

O(nlogn):归并、快排、堆排

O(n):桶排、计排、基排

回溯

属于不撞南墙不死心，一直干到头，中间加上些条件可以做到剪枝。

递归

玄学(汉诺塔我到现在都没弄明白),递归最主要的就是千万别人肉递归，最多递归1，2层。最主要的是找到terminator和drill down的条件

分治

先把大问题分成小问题，再解决子问题，再把子结果合并。(归并排序)就是个好🌰

贪心

贪心不太容易好证明是最优解，起码需要很强的数学功底。只能凭感觉，在我看来跟递归一样也是玄学。

滑动窗口

可以看成队列操作，满足条件入队列(移动右边指针)，不满足条件出队列(移动左边指针)，常见于求满足条件的连续子串问题

双指针

左右指针轮番动，直到满足条件为止

链表

增、删、换位置、逆序

栈、队列、数组

常常跟别的算法凑到一起

二分搜索

数据结构只要满足两段性就可以用二分，或者是处理过后的数组，比如前缀和，也可以用二分 。一定要满足两段性。

图

说白了图算法都是对节点的遍历。 DFS、BFS、拓扑排序、最短路径
